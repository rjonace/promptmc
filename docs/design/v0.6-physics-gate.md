# v0.6 — Physics safety gate

**Status:** Planned. Acceptance criteria: `AGENTS.md` §5.3.

## Goal

Deterministic pre-run validation that catches cell overlaps, unbounded
geometries, void cells, and tracking inconsistencies *before* OpenMC
runs — with structured, machine-routable failure explanations. This
gate is the safety boundary that later makes LLM generation (v0.8)
acceptable: every generated model must pass it before anything
executes.

## Design

### Be honest about what is tractable

General overlap detection in CSG is not analytically decidable for
arbitrary surface combinations. The design splits the work instead of
pretending otherwise:

- **Cheap, exact, pure Python:** unique IDs; every region references an
  existing surface; every cell has a fill (or is legitimately void);
  the root universe's outer boundary is `vacuum`/`reflective` (the
  anti-hang check); material densities/fractions sane; nuclide names
  well-formed.
- **Hard, approximate, delegated:** geometric overlaps and coverage
  gaps go to the existing `openmc_geometry_debug` tool (OpenMC's own
  geometry-debug mode, low particle count). The validator flags the
  structural problems exactly and routes the geometric ones — it does
  not reinvent a CSG solver, and its limits are documented loudly:
  this is pre-run input validation, not V&V or qualification.

### Structured failures (the part an agent consumes)

```python
@dataclass(frozen=True)
class ValidationIssue:
    code: str                          # "UNBOUNDED_GEOMETRY", "CELL_OVERLAP", ...
    severity: Literal["error", "warning"]
    message: str                       # human-readable
    location: str | None               # "cell 3", "surface 12"
    fix_hint: str | None               # "set boundary_type='vacuum' on surface 7"
```

`code` makes failures programmatically routable; `fix_hint` is what
the v0.8 repair loop injects back into a prompt. The vocabulary of
codes stays small and stable — it becomes an API.

### Tools and CLI

- **`openmc_trace_point(model, x, y, z)`** → the cells claiming that
  coordinate. Zero cells = coverage gap; two or more = overlap. The
  precise debugging tool the structural checks can't be.
- **`promptmc validate <geometry>`** extended to run the gate: prints
  structured issues, non-zero exit on any `error`.

### Performance constraint

The gate runs on every generation attempt in v0.8, so the cheap path
must be fast and deterministic — no OpenMC execution inside it.

## Testing

- Every cheap check gets a positive case (clean model passes) and a
  negative case (deliberately broken model yields exactly the expected
  `code`).
- **The v0.4 benchmarks are the regression suite:** parametrized over
  `ALL_BENCHMARKS`, the gate must return zero errors for every one.
- `trace_point`: one cell inside Godiva's sphere, zero outside.

## Open questions

- What counts as a *warning* vs an *error*? (E.g. a void cell with a
  bounded parent is legal but often a mistake.) The severity policy is
  the real design decision here, because the v0.8 repair loop trusts
  it.
- Should the gate auto-invoke `openmc_geometry_debug` when OpenMC is
  available, or only recommend it in the report? (Current lean:
  recommend; keep the gate side-effect-free.)
- Nuclide validation: well-formedness only (`U235`, `O16`), or check
  against a known-nuclide list — and if so, whose?
