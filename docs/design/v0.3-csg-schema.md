# v0.3 — CSG schema + serialization

**Status:** Shipped (v0.3.0 2026-06-07). Schema surface is
**experimental** until v0.4.

## Goal

A typed Pydantic model of OpenMC constructive solid geometry (CSG),
materials, and tallies that serializes to runnable OpenMC XML — the
foundation for the reference library (v0.4), the physics gate (v0.6),
and constrained generation (v0.8).

## Background

OpenMC geometry is built from surfaces, half-spaces, regions, cells,
and universes, normally via OpenMC's Python operator API
(`-sphere & +plane`). That API is expressive but not validatable or
generatable: there is no schema an LLM can be constrained against, and
errors surface at simulation time (sometimes as infinite hangs). v0.3
models CSG as data.

## Design

### Key decisions

- **Surfaces are a discriminated union.** Each surface type carries a
  `kind` literal (`"sphere"`, `"z-cylinder"`, …) used as the Pydantic
  discriminator. The model self-describes its variants, which is what
  makes it usable as an LLM constraint surface later.
- **Regions are an explicit tree, not a string.** `HalfSpace`,
  `Intersection`, `Union_`, `Complement` as recursive nodes. More
  verbose than OpenMC's operator magic, but validatable and
  generatable.
- **Validation at model time, not simulation time.** Cross-field
  `model_validator`s enforce unique IDs, no dangling surface
  references, and the anti-hang rule: the outermost boundary must be
  `vacuum`/`reflective`, never `transmission` (an unbounded geometry
  makes OpenMC track particles forever).
- **Dual-mode serialization (avoid double maintenance).** When OpenMC
  is importable, serialize *through* OpenMC objects
  (`.export_to_xml()`) so output always matches OpenMC's current
  schema. When absent, dump the Pydantic models to an intermediate dict
  and write XML with a small generic dict-to-XML utility — no parallel
  hand-maintained XML generator.
- **Schema separate from serialization.** Models import neither XML
  machinery nor OpenMC, so they can serve as pure JSON Schema for
  structured generation without dragging in heavy dependencies.
- **Optional IDs, auto-assigned.** Users (and models) may omit IDs;
  serialization assigns them, mirroring OpenMC's auto-numbering.
  Uniqueness is validated when IDs are provided.

### Module layout

```text
src/promptmc/geometry/
├── __init__.py          # curated, EXPERIMENTAL exports
├── primitives.py        # Surface union, region tree, Cell, Universe, GeometryModel
├── materials.py         # NuclideSpec, Material, MaterialsModel
├── tallies.py           # TallyFilter, Tally, TalliesModel (thin in v0.3)
└── xml_serializer.py    # model -> openmc objects -> XML, + dict fallback

src/promptmc/benchmarks/
├── _types.py            # Benchmark protocol: build() + EXPECTED_KEFF + bounds + SOURCE
├── godiva.py            # ICSBEP HEU-MET-FAST-001 (bare HEU sphere, vacuum boundary)
└── pwr_pin.py           # Mosteller PWR pin cell (reflective boundary)
```

The two seed benchmarks double as schema integration tests and as the
template for the v0.4 library.

## Testing

- **Property-based round-trips (Hypothesis):** generate valid models,
  serialize, re-parse, assert structure preserved. Comparison happens
  at the object level, not by string-diffing XML (OpenMC reorders
  attributes).
- **Golden benchmarks:** Godiva and the PWR pin build and serialize
  with the intended object graph. Simulation (k-eff) verification is
  deferred to v0.4 — it requires cross-section data CI doesn't have.
- **Conditional markers:** `requires_openmc` / `requires_openmc_data`
  keep the suite green when OpenMC or nuclear data are absent.

## Constraints

- Public schema surface marked experimental until v0.4 stabilizes it;
  after that it falls under the no-breaking-changes-in-minors rule.
- A model that validates must produce a runnable OpenMC input — that
  contract is the whole point of the schema.
